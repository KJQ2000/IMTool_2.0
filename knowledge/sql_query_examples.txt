====== EXAMPLE 01: COUNT CURRENT IN-STOCK ITEMS ======
User question:
How many items are currently in stock?
SQL:
SELECT COUNT(*) AS in_stock_count
FROM konghin.stock
WHERE stk_status = 'IN STOCK';
Reasoning:
Use the stock table and filter on the current availability status.

====== EXAMPLE 02: LIST CURRENTLY BOOKED STOCK ITEMS ======
User question:
Show all items that are currently booked.
SQL:
SELECT
    stk_id,
    stk_type,
    stk_pattern,
    stk_weight,
    stk_book_id,
    stk_last_update
FROM konghin.stock
WHERE stk_status = 'BOOKED'
ORDER BY stk_last_update DESC NULLS LAST, stk_id DESC;
Reasoning:
Booked stock is stored directly in the stock table with status BOOKED.

====== EXAMPLE 03: COUNT STOCK BY STATUS ======
User question:
Give me the stock count by status.
SQL:
SELECT
    COALESCE(stk_status, 'UNKNOWN') AS stock_status,
    COUNT(*) AS item_count
FROM konghin.stock
GROUP BY COALESCE(stk_status, 'UNKNOWN')
ORDER BY item_count DESC, stock_status;
Reasoning:
This groups the stock table by status and counts rows in each group.

====== EXAMPLE 04: TOP SELLING STOCK TYPES ======
User question:
What are the top selling stock types?
SQL:
SELECT
    COALESCE(stk_type, 'UNKNOWN') AS stock_type,
    COUNT(*) AS sold_item_count,
    ROUND(COALESCE(SUM(stk_profit), 0)::numeric, 2) AS total_profit_rm
FROM konghin.stock
WHERE stk_status = 'SOLD'
GROUP BY COALESCE(stk_type, 'UNKNOWN')
ORDER BY sold_item_count DESC, total_profit_rm DESC;
Reasoning:
Each sold stock row represents one sold item, so stock is the best source.

====== EXAMPLE 05: TOTAL SALES REVENUE THIS YEAR ======
User question:
What is the total sales revenue this year?
SQL:
SELECT
    COUNT(*) AS sale_count,
    ROUND(COALESCE(SUM(sale_price), 0)::numeric, 2) AS total_sales_revenue_rm
FROM konghin.sale
WHERE sale_sold_date >= DATE_TRUNC('year', CURRENT_DATE)::date
  AND sale_sold_date < (DATE_TRUNC('year', CURRENT_DATE) + INTERVAL '1 year')::date;
Reasoning:
The sale table stores header-level total sale_price values and sale dates.

====== EXAMPLE 06: MONTHLY SALES THIS YEAR ======
User question:
Show me monthly sales totals for this year.
SQL:
SELECT
    TO_CHAR(DATE_TRUNC('month', sale_sold_date), 'YYYY-MM') AS sale_month,
    COUNT(*) AS sale_count,
    ROUND(COALESCE(SUM(sale_price), 0)::numeric, 2) AS total_sales_rm
FROM konghin.sale
WHERE sale_sold_date >= DATE_TRUNC('year', CURRENT_DATE)::date
  AND sale_sold_date < (DATE_TRUNC('year', CURRENT_DATE) + INTERVAL '1 year')::date
GROUP BY DATE_TRUNC('month', sale_sold_date)
ORDER BY DATE_TRUNC('month', sale_sold_date)
LIMIT 12;
Reasoning:
Bucket sales by month using DATE_TRUNC on the sold date.

====== EXAMPLE 07: PENDING BOOKINGS WITH REMAINING BALANCE ======
User question:
Show pending bookings that still have remaining balance.
SQL:
SELECT
    book_id,
    book_cust_id,
    book_receipt_no,
    book_date,
    book_price,
    book_remaining,
    book_status
FROM konghin.booking
WHERE book_status = 'BOOKED'
  AND COALESCE(book_remaining, 0) > 0
ORDER BY book_date DESC NULLS LAST, book_id DESC;
Reasoning:
The booking table already stores the remaining balance and status.

====== EXAMPLE 08: BOOKING PAYMENT SUMMARY ======
User question:
Show the payment summary for each booking.
SQL:
SELECT
    b.book_id,
    b.book_cust_id,
    b.book_price,
    b.book_remaining,
    ROUND(COALESCE(SUM(bp.bp_payment), 0)::numeric, 2) AS total_paid_rm
FROM konghin.booking b
LEFT JOIN konghin.book_payment bp
    ON bp.bp_book_id = b.book_id
GROUP BY
    b.book_id,
    b.book_cust_id,
    b.book_price,
    b.book_remaining,
    b.book_date
ORDER BY b.book_date DESC NULLS LAST, b.book_id DESC;
Reasoning:
Join booking to book_payment and aggregate payment amounts by booking.

====== EXAMPLE 12: UNPRINTED BARCODE CANDIDATES ======
User question:
Show stock items that have not been printed for barcode export yet.
SQL:
SELECT
    stk_id,
    stk_barcode,
    stk_type,
    stk_pattern,
    stk_printed,
    stk_created_at
FROM konghin.stock
WHERE COALESCE(stk_status, '') = 'IN STOCK'
  AND COALESCE(NULLIF(TRIM(CAST(stk_printed AS TEXT)), ''), '0') IN ('0', 'FALSE', 'false')
ORDER BY stk_created_at DESC NULLS LAST, stk_id DESC;
Reasoning:
The stock table stores both barcode and print-state fields.

====== EXAMPLE 14: RECENT PURCHASES WITH SUPPLIER NAME ======

====== EXAMPLE 15: STOCK ITEMS FROM A PURCHASE CODE ======
User question:
Show all stock items from purchase code PUR-CODE-001.
SQL:
SELECT
    st.stk_id,
    st.stk_type,
    st.stk_pattern,
    st.stk_status,
    p.pur_code,
    p.pur_date
FROM konghin.stock st
JOIN konghin.purchase p
    ON p.pur_id = st.stk_pur_id
WHERE p.pur_code = 'PUR-CODE-001'
ORDER BY st.stk_id;
Reasoning:
The stock row links to purchase by stk_pur_id, while business users often know the purchase code.

====== EXAMPLE 18: SOLD STOCK PROFIT THIS MONTH ======
User question:
Show profit by sold stock item for this month.
SQL:
SELECT
    stk_id,
    stk_type,
    stk_pattern,
    stk_sell_date,
    ROUND(COALESCE(stk_profit, 0)::numeric, 2) AS profit_rm
FROM konghin.stock
WHERE stk_status = 'SOLD'
  AND stk_sell_date >= DATE_TRUNC('month', CURRENT_DATE)::date
  AND stk_sell_date < (DATE_TRUNC('month', CURRENT_DATE) + INTERVAL '1 month')::date
ORDER BY stk_profit DESC NULLS LAST, stk_id DESC;
Reasoning:
Profit is stored per sold stock row, so stock is better than sale for item-level profit.

====== EXAMPLE 20: SUPPLIERS WITH THE MOST PURCHASES ======
User question:
Which suppliers or salesmen have the most purchases?
SQL:
SELECT
    s.slm_id,
    s.slm_name,
    s.slm_company_name,
    COUNT(p.pur_id) AS purchase_count,
    ROUND(COALESCE(SUM(p.pur_total_amt), 0)::numeric, 2) AS total_purchase_rm
FROM konghin.salesman s
LEFT JOIN konghin.purchase p
    ON p.pur_slm_id = s.slm_id
GROUP BY s.slm_id, s.slm_name, s.slm_company_name
ORDER BY purchase_count DESC, total_purchase_rm DESC;
Reasoning:
Purchase records link back to salesman or supplier master data.

====== EXAMPLE 21: CUSTOMERS WITH COMPLETED SALES TOTALS ======
User question:
Which customers have the highest total sales value?
SQL:
SELECT
    c.cust_id,
    c.cust_name,
    COUNT(s.sale_id) AS sale_count,
    ROUND(COALESCE(SUM(s.sale_price), 0)::numeric, 2) AS total_sales_rm
FROM konghin.customer c
JOIN konghin.sale s
    ON s.sale_cust_id = c.cust_id
GROUP BY c.cust_id, c.cust_name
ORDER BY total_sales_rm DESC, sale_count DESC;
Reasoning:
Aggregate sale header totals by customer to rank customer sales value.
